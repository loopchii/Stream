# Ethics

Stream exists to make public media analysis inspectable, not to turn people into flattened scores.

The repository is strongest when it keeps three things true at once:

- the math is clear
- the data boundary is honest
- the human consequences stay visible

## Core commitments

### 1. Visible boundaries

The repository should never blur the line between:

- synthetic teaching data
- public metadata analysis
- private or proprietary systems that are not part of this repo

If a surface cannot support a stronger claim, the wording should step down instead of pretending certainty.

### 2. Human consequences first

Representation analysis is not just a technical exercise. The patterns in this repo connect to who gets seen, heard, funded, hired, or overlooked.

That means the interface and documentation should not treat bias as trivia. They should help readers understand:

- what a metric is showing
- what it is not showing
- why the result matters to people
- what a responsible next question would be

### 3. No certainty theater

Not every measurable pattern deserves a hard conclusion.

Contributors should avoid:

- overstating weak evidence
- hiding missing labels or thin coverage
- turning directional findings into universal claims
- using technical language to make a soft conclusion sound stronger than it is

The repository should earn trust by being precise, not by sounding absolute.

### 4. Human decision responsibility

Stream can support analysis. It should not be positioned as a substitute for judgment in high-impact decisions.

The repository is not a stand-alone decision maker for:

- hiring
- casting
- admissions
- credit
- insurance
- medical treatment
- legal judgment

If someone wants to use outputs from Stream in a consequential process, the output should be treated as one input among several, with a responsible human still accountable for the final decision.

### 5. Reproducibility over performance

The repo should favor work that can be inspected and rerun over work that merely sounds advanced.

That means:

- documenting assumptions
- exposing source gaps
- keeping calculations traceable
- making it easy for others to challenge a result

## Contributor questions

Before adding a new visual, metric, or narrative sentence, ask:

1. What does this help someone understand?
2. What evidence actually supports it?
3. What limitation needs to stay visible beside it?
4. Could this framing make a human consequence easier to ignore?
5. Would a skeptical reader be able to verify the claim from the repo?

If the answer to the last question is no, the claim needs to be weaker or the evidence needs to be better.

## Responsible language

The tone of the repository should feel:

- rigorous
- readable
- open to challenge
- aware of consequence

It should not feel:

- grandiose
- manipulative
- clinical about harm
- falsely neutral when a result clearly matters

## Reporting concerns

If you see a claim, visual, or data choice that feels ethically off:

- open an issue
- point to the exact file, chart, or sentence
- explain whether the concern is about harm, overreach, missing uncertainty, or misuse risk

The right move is not to bury the concern. The right move is to make the boundary clearer.
